
このモジュールは既存のSaferOQLモジュールのSQL版です。SQLダイレクトに使用するため全てのSQLの機能を利用可能です。OQL特有の制限や方言に振り回される必要が無い一方でアソシエーションテーブルのJOINは明示的に記述する必要がある点には注意してください。


This module is the SQL version of the existing SaferOQL module. Since it operates directly on SQL, you can utilize the full range of SQL functionality. While you are not constrained by OQL-specific limitations or dialects, please note that you must explicitly write JOINs for association tables.




2WaySQLの変換機構は日本のオープンソースプロジェクトでるDBFluteのランタイムを使用していま。そのため2WaySQL中のパラメータには’pmb.’というプリフィクスが必ずに次要になります。
下記のようになります。
select
	"name","lastlogin","blocked","active","isanonymous"
from
	"system$user"
/*BEGIN*/
where
	/*IF pmb.Name != null*/
	"name" like  '%' || /*pmb.Name*/'Admin' || '%'
	/*END*/
	/*IF pmb.Blocked != null*/
	and "blocked" = /*pmb.Blocked*/'false'
	/*END*/
	/*IF pmb.Active != null*/
	and "active" = /*pmb.Active*/'true'
	/*END*/
	/*IF pmb.IsAnonymous != null*/
	and "isanonymous" = /*pmb.IsAnonymous*/'false'
	/*END*/
/*END*/


The 2WaySQL conversion mechanism utilizes the runtime of DBFlute, a Japanese open-source project. Consequently, the prefix "pmb." is mandatory for parameters within the 2WaySQL.
It looks like this:
```
select
	"name","lastlogin","blocked","active","isanonymous"
from
	"system$user"
/*BEGIN*/
where
	/*IF pmb.Name != null*/
	"name" like  '%' || /*pmb.Name*/'Admin' || '%'
	/*END*/
	/*IF pmb.Blocked != null*/
	and "blocked" = /*pmb.Blocked*/'false'
	/*END*/
	/*IF pmb.Active != null*/
	and "active" = /*pmb.Active*/'true'
	/*END*/
	/*IF pmb.IsAnonymous != null*/
	and "isanonymous" = /*pmb.IsAnonymous*/'false'
	/*END*/
/*END*/
```



2Way-SQLファイルは
```
AppProjectRoot\resources\sql\Stock\SEL_Stock_Inventory.sql
```
という感じでresources配下にsql(全て小文字)というフォルダを作って格納します。モジュール名でサブフォルダを作ってさらに機能で分けて格納する事をお勧めします。	
## エンコーディングは必ずUTF-8(BOM無し)として下さい。
## ファイル名の先頭はCRUDに合わせて「INS_」「SEL_」「UPD_」「DEL_」として下さい。
## MendixがID値を自動採番するアーキテクチャの制約上、この機能でinsert出来るのはアソシエーションエンティティのみです。
## MxCloud等のLinux系環境ではファイルシステムは大文字小文字を区別するのでフォルダ＆ファイル名の指定は要注意です。
## 本モジュールではDBFluteランタイムは2WaySQLをパースして標準SQLに変換するためにだけ使用しているため、それ以外のDBFluteの機能には対応していません。

Store 2Way-SQL files under the `resources` directory in a folder named `sql` (all lowercase), for example:
```
AppProjectRoot\resources\sql\Stock\SEL_Stock_Inventory.sql
```
It is recommended to organize files by creating subfolders based on module names and further categorizing them by function.
Ensure the file encoding is UTF-8 (without BOM).
Prefix filenames with "INS_", "SEL_", "UPD_", or "DEL_" to correspond with CRUD operations.
Due to Mendix's architecture regarding automatic ID generation, only association entities can be inserted using this function.
In Linux-based environments such as MxCloud, the file system is case-sensitive; therefore, exercise caution when specifying folder and file names.
In this module, the DBFlute runtime is used solely to parse 2WaySQL and convert it into standard SQL; therefore, other DBFlute features are not supported.


Mendixアプリの内部データベースのアクセスだけでなく外部データベースへのアクセスも可能です。
外部データベースにアクセスする場合はアプリ起動時に AfterStartUpの中からRegisterExternalDataSourceアクションで外部データソースを登録しておく必要があります。  
その上で下記の様に2WaySQL中に外部データソースディレクティブを指定してください。
```
-- It's the external data source directive
-- @YourExtDataSourceName@ 
select
 name, description
from
 your_external_table
```


You can access not only the Mendix app's internal database but also external databases.
To access an external database, you must register the external data source using the `RegisterExternalDataSource` action within the `AfterStartUp` event when the app starts.
Then, specify the external data source directive within the 2WaySQL as shown below.
```
-- It's the external data source directive
-- @YourExtDataSourceName@ 
select
 name, description
from
 your_external_table
```

注意：外部データソースへのアクセスの際にはトランザクション制御はオートコミットとなります。
また使用するJDBCドライバはStudioProのJava依存性定義を使ってダウンロードする必要があります。

Note: Transaction control defaults to auto-commit when accessing external data sources.

A simple sample is included in the `_Sample` folder, so we recommend checking it first.



-----------------------------------------------------------------------------------------------
このモジュールは既存のSaferOQLモジュールのSQL版です。
SQLをダイレクトに使用するため全てのSQLの機能を利用可能ですし、SaferOQL同様のテスト機能によってSQLとドメインモデルの乖離を一括検出可能です。
更に外部データベースへの接続機能も追加されているので内部DBと同じように外部データベースにもアクセス可能です。
大規模データ処理のためのバッチフェッチループ機能もあるため、大量データ処理のための基盤としてもお勧めです。

This module is the SQL-based counterpart to the existing SaferOQL module.
Because it utilizes SQL directly, it supports the full range of SQL functionality; furthermore, it offers testing capabilities similar to SaferOQL, enabling the batch detection of discrepancies between the SQL and the domain model.
It also includes features for connecting to external databases, allowing you to access them just as you would an internal database.
With a built-in batch fetch loop for handling large datasets, it is highly recommended as a foundation for processing massive amounts of data.
